⛏ Minecraft Server Sync

Guia completo de instalação e uso — NeoForge 1.21.1 + GitHub

O que você quer fazer?

🎮 Parte 1 — Configurar o cliente

Faça isso para conseguir entrar no servidor e jogar.

Passo 1 — Instalar o Java 21

O Java é necessário para rodar o NeoForge.

Precisa ser o Java 21. Versões mais antigas não funcionam.
BAIXAR

Acesse adoptium.net e baixe o Temurin 21 (LTS) para Windows.

adoptium.net — Temurin 21
INSTALAR

Execute o instalador. Se aparecer a opção "Add to PATH", marque ela — é importante.

VERIFICAR

Abra o Prompt de Comando (tecla Windows → digite cmd → Enter) e rode:

java -version

Deve aparecer openjdk version "21.x.x".

Passo 2 — Instalar o NeoForge no Minecraft

O NeoForge é o mod loader — sem ele o Minecraft não consegue carregar os mods. Você vai receber um arquivo chamado neoforge-21.1.x-installer.jar.

EXECUTAR O INSTALLER

Dê duplo clique no arquivo neoforge-...-installer.jar. Uma janela vai abrir:

  • Selecione "Install client"
  • O caminho do Minecraft já vem preenchido automaticamente — não precisa mudar
  • Clique em OK e aguarde terminar
VERIFICAR NO LAUNCHER

Abra o launcher do Minecraft. No seletor de versões (canto esquerdo inferior), deve aparecer um perfil chamado NeoForge 1.21.1-xx.x.x. Selecione ele.

Se o launcher não abrir ao dar duplo clique no .jar, clique com botão direito → Abrir com → Java(TM) Platform SE Binary.

Passo 3 — Instalar os mods e configs

Você vai receber um arquivo .zip com duas pastas dentro: mods e config. Extraia o zip e siga os passos abaixo.

INSTALAR OS MODS

Pressione Windows + R, cole o caminho abaixo e pressione Enter:

%appdata%\.minecraft\mods

Copie todos os arquivos .jar da pasta mods do zip para dentro dessa pasta.

Se já tiver mods de outro modpack, remova tudo antes de colocar os novos para evitar conflitos.
INSTALAR AS CONFIGS

Pressione Windows + R, cole o caminho abaixo e pressione Enter:

%appdata%\.minecraft\config

Copie o conteúdo da pasta config do zip para dentro dessa pasta.

Se perguntar se quer substituir arquivos, clique em Sim para todos.
TESTAR

Abra o Minecraft, selecione o perfil NeoForge 1.21.1 e clique em Jogar. Se carregar sem erros, está tudo certo.

Para entrar no servidor, use o IP que o dono te passar.

🖥️ Parte 2 — Configurar o servidor

Configure o servidor local e a sincronização do mundo com o GitHub.

Passo 1 — Instalar o Git

O Git mantém o mundo sincronizado entre os jogadores via GitHub.

BAIXAR

Acesse git-scm.com e baixe o instalador para Windows.

git-scm.com/download/win
INSTALAR

Execute o instalador. Pode clicar em Next em todas as telas — as opções padrão funcionam.

VERIFICAR

Abra o Prompt de Comando e rode:

git --version

Deve aparecer git version 2.x.x.

Passo 2 — Baixar os arquivos do servidor

CRIAR A PASTA

Crie uma pasta para o servidor, por exemplo C:\minecraft-server\. Abra essa pasta no Explorer, clique com botão direito no espaço vazio e selecione "Abrir no Terminal" ou "Git Bash Here".

CLONAR

Cole o comando abaixo no terminal (use a URL do repositório):

git clone https://github.com/USUARIO/REPOSITORIO.git .
O ponto no final é importante — faz os arquivos aparecerem direto na pasta atual.

Depois disso a pasta vai ter estes arquivos:

minecraft-server/ ├── sync.exe ← programa principal ├── server_setup.exe ← setup inicial ├── config.example.json ← modelo de configuração ├── status.json ← mostra quem está jogando └── server/ ← pasta do servidor (ainda vazia)

Passo 3 — Colocar os mods no servidor

COPIAR OS MODS

Copie os arquivos .jar do modpack para a pasta server\mods\ (crie a pasta se não existir):

minecraft-server/ └── server/ └── mods/ ← cole aqui os .jar dos mods
Pode colocar todos os mods incluindo os client-only — o server_setup.exe vai remover automaticamente os que não funcionam no servidor.

Passo 4 — Instalar o NeoForge no servidor

Use o mesmo arquivo neoforge-...-installer.jar — desta vez ele vai instalar no servidor.

COLOCAR O INSTALLER

Coloque o arquivo neoforge-...-installer.jar dentro da pasta server\.

server/ ├── mods/ └── neoforge-21.1.x-installer.jar ← aqui
RODAR O SETUP

Abra o terminal na pasta do servidor e rode:

server_setup.exe

Isso vai:

  1. Instalar o NeoForge no servidor (~500 MB, pode demorar)
  2. Remover os mods client-only de server\mods\
  3. Criar o eula.txt e o server.properties

Passo 5 — Configurar

CRIAR config.json

Na pasta do servidor, copie o arquivo config.example.json e renomeie a cópia para config.json.

EDITAR

Abra o config.json com o Bloco de Notas e preencha os campos marcados:

{
  "player_name": "SeuNomeAqui",
  "server_path": "./server",
  "start_command": [
    "java",
    "@user_jvm_args.txt",
    "@libraries/net/neoforged/neoforge/21.1.218/win_args.txt",
    "nogui"
  ],
  "backup_interval_minutes": 15,
  "rcon": {
    "host": "127.0.0.1",
    "port": 25575,
    "password": "SENHA_AQUI"
  },
  "git": {
    "remote_url": "https://github.com/USUARIO/REPOSITORIO.git",
    "branch": "main"
  }
}
  • Troque SeuNomeAqui pelo seu nome ou apelido
  • Troque SENHA_AQUI pela senha do RCON (a mesma que está em server\server.properties)
  • Troque a URL do GitHub pela URL correta do repositório
Não mexa nos outros campos. Salve o arquivo depois de editar.

Passo 6 — Inicializar o repositório (só na primeira vez)

SETUP

Abra o terminal na pasta do servidor e rode:

sync.exe setup

Isso conecta a pasta ao GitHub e faz o primeiro upload.

Se aparecer uma janela pedindo login no GitHub, entre com sua conta normalmente.

Uso do dia a dia

Para subir o servidor

sync.exe start

Verifica se alguém está jogando, baixa o mundo mais recente do GitHub e sobe o servidor. Seus amigos conectam pelo seu IP local.

Ver quem está jogando

sync.exe status

Parar o servidor

Digite /stop no console do servidor ou pressione Ctrl+C no terminal. O mundo é salvo e enviado pro GitHub automaticamente.

Se o servidor travou e o lock ficou preso

sync.exe force-release
Use só se o servidor não está mais rodando. Nunca use enquanto alguém estiver jogando.

Referência de comandos

ComandoO que faz
sync.exe startVerifica lock, faz pull, sobe o servidor, backups automáticos, push ao fechar
sync.exe statusMostra quem está jogando ou se está livre
sync.exe force-releaseForça liberação do lock (usar só se travou)
sync.exe setupInicializa git e faz o primeiro push (só na primeira vez)
server_setup.exeInstala NeoForge, limpa mods client-only, cria eula e server.properties

Regra de ouro

Sempre use sync.exe start para iniciar o servidor.

Nunca abra pelo run.bat diretamente — o mundo não será salvo no GitHub e outros jogadores não vão conseguir jogar depois.
